## 🎯 Идея 1 — Operational Signal Layer (OSL): Physical → Financial mapping — ✅ РЕАЛИЗОВАНО в v0.9 (июнь 2026)

> **Это центральная идея, которую запросил пользователь.** Производство нефти / стали / меди / GMV → выручка → EBITDA → PD.

> [!check] **Реализовано в v0.9 (июнь 2026)** как слой L1.5: OSL для 7 отраслей (`osl_*.py`) + общий `osl_common.py`, conformal-интервалы (in-sample), `fetch_macro_state.py` (4 живых макрофида). Out-of-sample conformal — в планах (нет независимых 9M actuals). Было: «план Фаза 1.5, июль–август 2026».

### Что это

Для каждой отрасли строится **функция**: physical signal × price → financial outcome. Это даёт **2–3 месяца форы** перед публикацией МСФО — операционные данные обновляются ежемесячно и часто **до** финансовых публикаций.

| Отрасль | Operational Signal | Формула revenue (приближённо) |
|---|---|---|
| **Нефтегаз** | Добыча (млн т / млн т н.э.) | `Q × (Urals × курс − НДПИ × курс)` |
| **Металлургия (чёрная)** | Производство стали (млн т) | `Q × (FOB ЧМ × курс)` |
| **Металлургия (цветная)** | Объём меди / никеля (тыс. т) | `Q × LME × курс` |
| **Металлургия (драгметаллы)** | Унции золота | `Q × LBMA × курс` |
| **Розница (e-com)** | GMV / транзакции эквайринга | `GMV × take_rate` |
| **Розница (offline)** | LFL-трафик ТЦ | `Traffic × средний чек` |
| **Энергетика** | Выработка кВтч | `Q × тариф` |
| **Химия (удобрения)** | Производство аммиака | `Q × FOB Балтика × курс` |
| **Фарма** | Упаковки в розничных продажах | `Packs × средняя цена` |
| **ОИВ** | Налоговые поступления нефтегазовых компаний | По формуле НДПИ |

### Релевантные публикации (2024–2025)

- **[arXiv:2508.18289](https://arxiv.org/abs/2508.18289)** — Data-driven models for production forecasting in petroleum reservoirs (08.2025). Supervised learning + neural networks для прогноза производственных параметров.
- **[arXiv:2508.14078](https://arxiv.org/abs/2508.14078)** — Out-of-Sample Hydrocarbon Production Forecasting with **Inductive Conformal Prediction** (08.2025). Даёт калиброванные интервалы доверия.
- **[arXiv:2409.16482](https://arxiv.org/abs/2409.16482)** — Generative AI-driven forecasting of oil production (09.2024). Использует TimeGrad и Informer для long-sequence ts.
- [Frontiers in Energy Research (2024)](https://www.frontiersin.org/journals/energy-research/articles/10.3389/fenrg.2024.1369606/full) — deep learning для ультра-глубоких трещиновых пластов.

### Источники operational signals

| Канал | Что берём | Лаг |
|---|---|---|
| Минэнерго / ФТС | Добыча и экспорт нефти | 2–3 нед |
| Worldsteel / WSA | Производство стали | 1 мес |
| LME / SMM | Цены меди, никеля | 1 день |
| LBMA / WGC | Цены золота | 1 день |
| Росстат | Производство по ОКВЭД | 1 мес |
| FRED Goods exports | Экспорт по группам | 1 мес |
| **СПУТНИКОВЫЕ ДАННЫЕ** | Активность портов, рудников, складов | 1–7 дней |

### Конкурентное преимущество

**2–3 месяца форы** перед раскрытием МСФО → решения по риск-аппетиту опережают рейтинговые действия АКРА/Эксперт РА.

## 🛰️ Идея 2 — Satellite-augmented macro nowcaster

### Что это

Использовать публичные спутниковые снимки для nowcasting экономической активности. Покрытие России — отличное (Sentinel-2, Landsat 8/9, VIIRS). Что меряем:

- **Активность портов** (Новороссийск, Усть-Луга, Владивосток, Находка) — счёт судов на якоре + время разгрузки
- **Промышленные центры** — Норильск (медь), Магнитогорск (сталь), Сургут (НГ) — индекс активности
- **Сельхоз-регионы** — NDVI (растительность), прогноз урожая
- **Ночные огни** (VIIRS Day/Night Band) — индекс региональной активности

### Релевантные публикации

- **[A View Across the Skyline (SSRN, 01.2025)](https://papers.ssrn.com/sol3/Delivery.cfm/5101867.pdf)** — комбинирует daytime satellite + electricity + gas consumption, AFF + CNN.
- **[arXiv:2407.12953](https://arxiv.org/abs/2407.12953)** — Rural marketplaces from satellite (Эфиопия, аналог можно сделать для РФ-региональных рынков).
- **[Big brothering the economy (Royal Statistical Society, 2024)](https://academic.oup.com/jrsssa/advance-article/doi/10.1093/jrsssa/qnae157/8078567)** — **Port Saturation Index** для nowcasting GDP и trade flows.
- [SpaceKnow](https://spaceknow.com/products/economic/) — коммерческая платформа (можно как референс).

### Конкурентное преимущество

Когда **порты Новороссийска перегружены** → экспорт металлов / нефти в этом квартале будет высоким → выручка ↑. Это видно в реальном времени через спутник, **до публикации ФТС-статистики** (которая выходит с лагом 1–1.5 мес).

## 🤖 Идея 3 — Multi-Agent LLM Pipeline для L0 (заменяет одиночный промпт) — ✅ РЕАЛИЗОВАНО в v0.9 (июнь 2026)

> [!check] **Реализовано в v0.9 (июнь 2026):** 5-агентный `orchestrator.py` (реальный код, не план) + RAG (`agents/rag/`). Совместно с Идеей 11. Было: «план Фаза 0→1, июнь 2026».

### Что это

Сегодня наш L0 — это **один большой промпт**. Свежие модели предлагают **multi-agent pipeline**:

```
Новость → 
  ├─ Агент-Классификатор (категория шока)
  ├─ Агент-Контекст (RAG по 32+ источникам всех отраслей)
  ├─ Агент-Бэк-тест (поиск исторических аналогов)
  ├─ Агент-Импакт (применение карты L1/L2/L3)
  └─ Агент-Сводка (компиляция всех в один отчёт)
```

### Релевантные публикации

- **[arXiv:2510.23032](https://arxiv.org/html/2510.23032v1)** — **P1GPT**: Multi-Agent LLM Workflow для Multimodal Financial Information. Архитектура:
  1. Input Layer — парсит данные
  2. Planning Layer — task decomposition
  3. Analysis Layer — Intelligent Specialized Agents (фундаментальный, технический, новостной, секторальный)
  4. Integration Reasoning Layer — fusion cross-modal outputs
- **[arXiv:2502.00415](https://arxiv.org/abs/2502.00415)** — **MarketSenseAI 2.0** с Chain-of-Agents (CoA) — granular handling of 10-Q/10-K, earnings calls.
- **[arXiv:2508.11152](https://arxiv.org/html/2508.11152v1)** — AlphaAgents для Equity Portfolio Construction.

### Конкурентное преимущество

Один LLM-промпт ломается на сложных композитных шоках (типа «Иран + санкции + Brent shock»). Multi-agent — каждый агент специализируется и **передаёт промежуточный результат** дальше с метаданными → меньше галлюцинаций, больше прослеживаемости.

## 🔍 Идея 4 — Causal Discovery (LPCMCI) на отраслевых индикаторах

### Что это

Сейчас наша 7×7 матрица — **экспертная**: H/M/L заполнены руками. Можно **открыть причинные связи статистически** из 8-летних данных (2017–2025) с помощью алгоритма **LPCMCI** (Latent Peter-Clark Momentary Conditional Independence) с **Gaussian Process Distance Correlation (GPDC)**.

Алгоритм находит:
- Прямые причинные связи (A → B, не A ↔ B)
- С учётом **скрытых переменных** (latent confounders) — это критично для макро (есть скрытые «настроения», KС-ожидания и т.д.)
- Лаги (A в момент t → B в момент t+k)

### Релевантные публикации

- **[arXiv:2509.07036](https://arxiv.org/abs/2509.07036)** — **Structural Causal Modelling and Uncertainty-Aware Forecasting for Economic Indicators** (09.2025). Применяет LPCMCI + GPDC к US GDP, growth, inflation, unemployment 1970–2021. **Точно такая же методология применима к нашим 7×7 + макро-индикаторам.**
- **[arXiv:2505.16308](https://arxiv.org/pdf/2505.16308)** — **CAIFormer**: Causal-Aware Time Series Forecasting (05.2025).
- **[arXiv:2312.17375](https://arxiv.org/abs/2312.17375)** — Causal Discovery in Financial Markets для Nonstationary Time-Series.

### Конкурентное преимущество

Заменяет «экспертные оценки» количественными **причинными** связями. Кредитный комитет может видеть **statistically significant edge** в матрице вместо «эксперт сказал H».

## 📊 Идея 5 — Foundation Time Series Models (TimesFM / Chronos-2 / Moirai)

### Что это

Pre-trained foundation models для time series — аналог GPT-4 для текста, но для рядов. Главные:

| Модель | Автор | Параметры | Особенность |
|---|---|---|---|
| **TimesFM** | Google | 200M | Decoder-only, **100B real-world time-points**, zero-shot |
| **Chronos / Chronos-2** | Amazon | 8M–710M | Tokenized time series, лидер GIFT-Eval (2025) |
| **Moirai / Moirai-MoE** | Salesforce | 14M–311M | Encoder-only, cross-domain |
| **Lag-Llama** | (open community) | 2.5M | Decoder-only, lags as covariates, probabilistic |
| **MOMENT** | (open) | 380M | Self-supervised pre-training |
| **Time-MoE** | — | — | Mixture of experts |

Все — **zero-shot prediction** возможен без обучения. Идеально для коротких русских рядов (мало данных для своего обучения).

### Релевантные публикации

- **[arXiv:2511.18578](https://arxiv.org/html/2511.18578v1)** — **Re(Visiting) Time Series Foundation Models in Finance** (11.2025). Pre-trained TSFMs дают «substantial forecasting and economic gains» в финансах.
- **[arXiv:2505.11163](https://arxiv.org/html/2505.11163v1)** — **Foundation Time-Series AI Model for Realized Volatility Forecasting** (05.2025).
- **[arXiv:2310.08278](https://arxiv.org/abs/2310.08278)** — Lag-Llama (open-source).

### Конкурентное преимущество

Zero-shot прогноз для **коротких рядов** (Россия после 2022 — мало данных). Не нужно обучать с нуля — берём pre-trained и сразу получаем прогноз.

### Что это

## 🧠 Идея 8 — Kolmogorov-Arnold Networks (KAN) для интерпретируемости

### Что это

KAN — **новейшая архитектура** (середина 2024), альтернатива классическому MLP. Преимущества:
- **Меньше параметров** (часто в 10–100× меньше при сравнимой точности)
- **Интерпретируемость** — можно вывести функциональные формы (а не только feature importance, как XGBoost)
- **Хорошо работает на time series**

KAN внутри L3 → можно сказать «КС влияет на ΔPD сегмента ФЛ.Массовый по такой-то нелинейной функции», а не просто «это сильный фичей».

### Релевантные публикации

- **[arXiv:2406.02496](https://arxiv.org/abs/2406.02496)** — KAN for Time Series: T-KAN и MT-KAN с **2 слоями × 5 нейронами** дают сравнимые результаты с MLP-baselines.
- **[arXiv:2504.16432](https://arxiv.org/html/2504.16432v1)** — **iTFKAN**: Interpretable Time Series Forecasting with KAN (04.2025).
- **[arXiv:2406.02486](https://arxiv.org/abs/2406.02486)** — **Temporal KAN Transformer** для finance.
- **[arXiv:2601.18837](https://arxiv.org/html/2601.18837v1)** — **HaKAN** (Hahn polynomials).

### Конкурентное преимущество

Стандарт CCAR — линейные модели + интерактивные термы. KAN даёт **нелинейные функциональные формы с интерпретацией**. Это решающее преимущество для регуляторного приёма.

## 🎯 Идея 9 — Causal-Aware Time Series Forecasting (CAIFormer)

### Конкурентное преимущество

Трансформеры мощные, но «black box». CAIFormer — попытка совместить мощь и понятность.

## 🔤 Идея 11 — News Sentiment Embeddings + RAG — ✅ РЕАЛИЗОВАНО в v0.9 (июнь 2026)

> [!check] **Реализовано в v0.9 (июнь 2026):** RAG в `agents/rag/*.py` (`init_db`, `embeddings`, `find_analogs`, `index_news`). Эмбеддер по умолчанию TF-IDF (всегда доступен), опционально e5-small (`multilingual-e5-small`, 384d) через env `RADAR_RAG_USE_ST` с graceful fallback. Индексация: `index_single` (UPSERT) + `index_all`.

### Что это

### Релевантные публикации

- **[arXiv:2507.01970](https://arxiv.org/abs/2507.01970)** — News Sentiment Embeddings for Stock Price Forecasting (07.2025).
- **[arXiv:2502.10990](https://arxiv.org/html/2502.10990v1)** — **FinMTEB**: Finance Massive Text Embedding Benchmark. **FinBERT обходит BERT на 15.6%; Fin-E5 обходит e5-mistral-7b-instruct на 4.5%.**
- **[arXiv:2512.08088](https://arxiv.org/abs/2512.08088)** — Adaptation of Embedding Models to Financial Filings via LLM Distillation (12.2025).

### Конкурентное преимущество

Бэк-тест становится **автоматическим**: новость → ближайшие исторические аналоги → их фактические эффекты. Никакого ручного поиска похожих событий.

## 🌍 Идея 12 — Counterfactual + Treatment Effect через DoWhy/EconML

### Что это

Применить **causal ML** для оценки treatment effect конкретного события:
- Что было бы с PD сегмента, **если бы не было** иранского кризиса?
- Какой **net effect** на ΔEBITDA отрасли от санкций ЕС-19?

Это `synthetic control` + `double ML`.

### Релевантные публикации

- **[Causal Inference for Banking, Finance, and Insurance](https://arxiv.org/pdf/2307.16427)** — обзорная статья.
- **[DoWhy/EconML](https://github.com/py-why/dowhy)** — open-source Python.
- **[Modern Causal Inference](https://medium.com/causal-inference-methods-models-and-applications/modern-causal-inference-methods-models-and-applications-preface-2a48342501c9)** — серия методов.

### Конкурентное преимущество

Стандарт CCAR не отвечает на «что было бы, если бы не». Causal ML — отвечает. Это даёт банку возможность **формально доказать ценность** своих риск-решений.

## 🏆 Топ-3 для немедленной интеграции в РАДАР

Из 12 идей — рекомендую внедрить **в первую очередь** эти три:

### 🥇 1. **Operational Signal Layer (OSL)** — Идея 1 — ✅ РЕАЛИЗОВАНО в v0.9 (июнь 2026)
**Почему:** прямо отвечает на запрос пользователя; **2–3 месяца форы** перед МСФО — главное конкурентное преимущество; не требует новых данных (всё открыто).

**Что делать:**
- Добавить **новый слой L1.5** в архитектуру: между Macro State и Industry Spillover
- Для 7 отраслей создать **OSL-функции**: physical signal × price → revenue → EBITDA → PD-pool
- Источники: Минэнерго, Worldsteel, LME/SMM, Росстат, ФТС

**Срок:** 60–80 часов. Оптимально — Фаза 1.5 (между Фазой 1 и Фазой 2).

### 🥈 2. **Multi-Agent LLM Pipeline для L0** — Идея 3 + 11 — ✅ РЕАЛИЗОВАНО в v0.9 (июнь 2026)

**Почему:** L0 сейчас — **bottleneck** (один промпт = высокий риск галлюцинации); multi-agent с RAG по нашим источникам + бэк-тест на исторических аналогах = **новый стандарт качества анализа**.

**Что делать:**
- Декомпозировать наш существующий промпт на **5 агентов** (P1GPT-style)
- Добавить **News Embeddings + RAG** (Идея 11): встраивание заголовков → ANN-поиск аналогов в истории
- Использовать **FinBERT / Fin-E5** для русского — Anthropic Claude эмбеддинги тоже подходят
- Все агенты — Claude Code (один runtime)

**Срок:** 30–40 часов (на Фазе 0 → 1).

### 🥉 3. **Causal Discovery (LPCMCI)** + **KAN** в L3 — Идеи 4 + 8

**Почему:** заменяет «экспертные» H/M/L количественными причинными связями (LPCMCI) + интерпретируемые модели вместо XGBoost (KAN). Это решающий аргумент для внутренней методологической защиты.

## Риски и ограничения свежих моделей

> [!warning] Не всё, что свежо — лучше

1. **Foundation TS Models (TimesFM, Chronos)** — pre-trained на западных данных. Могут плохо работать на РФ-специфике (КС-режимы 21% — outlier для модели). **Использовать с осторожностью**, проверять out-of-distribution.
2. **GNN на supply-chain** — требуют **bilateral exposure data**, которой у нас нет (открытые данные не дают экспозицию банка). **Откладываем до Фазы 4**.
3. **KAN** — новая архитектура, мало production-прецедентов. Использовать как exploratory, параллельно сохранять XGBoost-baseline.
4. **Multi-Agent LLM** — дорогие (много токенов). При большом объёме новостей нужен бюджет на инференс. **Использовать селективно** — только для шоков силы M+ (≥30/100).
5. **Causal Discovery (LPCMCI)** — статистически шумный для коротких выборок. Нужно ≥ 200 наблюдений per variable. У нас 96 месяцев = окей, но граничное условие.

## Что не вошло в топ-12 (но видел и оценил)

- **Bayesian Structural Time Series** (Google CausalImpact) — для оценки эффекта одиночного события. Полезно, но решает узкую задачу.
- **Variational Autoencoders** для anomaly detection в ts — есть в RiskLabs.
- **Differential Privacy** — для работы с внутренними данными банка с защитой PII. Релевантно фазе 4.
- **Federated Learning** — при объединении с другими банками. Не сейчас.
- **Sparse Identification of Nonlinear Dynamics (SINDy)** — открывает уравнения движения системы. Очень cool, но требует чистых данных.
