# 📘 Макроэкономика и финансы — для IT-специалиста

Документ переводит финансовые понятия на язык, понятный программисту. Цель — чтобы IT-специалист, открыв код Радара, понимал контекст того, что моделирует.

## 1. Что такое макроэкономика — в коде

Макроэкономика — это **глобальное состояние системы** (страна / мир), которое влияет на **локальные сегменты** (отрасли, компании, домохозяйства). Это как:

```python
class GlobalState:
    inflation: float          # рост цен
    interest_rate: float       # ключевая ставка ЦБ
    exchange_rate: float       # USD/RUB
    oil_price: float           # Brent
    budget_deficit: float

class Industry:
    state: GlobalState         # ссылка на macro
    revenue: float             # зависит от state
```

При изменении `GlobalState` — все Industry/Company пересчитывают свои метрики. Это и есть **transmission mechanism** (передача шока).

## 2. Ключевая ставка ЦБ (КС) — главный «масштабный коэффициент»

**Что:** процент, под который ЦБ кредитует коммерческие банки. На 25.04.2026 — 14.5%.

**Зачем нужна:** регулирует **стоимость денег** в экономике. Высокая КС → дорогие кредиты → меньше инвестиций → меньше инфляции, но меньше роста ВВП.

**Аналогия из IT:** КС — это **rate limiter** для всей экономики. Поднял ставку — система замедляется. Опустил — разгоняется.

**Что меняется при ↑ КС:**
- ↑ ставки по депозитам (банки могут платить больше)
- ↑ ставки по кредитам (потребкредит, ипотека)
- ↓ спрос на ипотеку → ↓ цены недвижимости
- ↓ спрос на потребкредиты → ↓ розничные продажи
- ↑ доходность ОФЗ (госбумаг) → отток из акций в облигации

**Где видно в Радаре:**
- В коде OSL — `KS_REGIME` (нормальный ≤10%, умеренный 10-18%, острый >18%)
- В таксономии шоков — категория **4. Ставка ЦБ** с подкатегориями 4.1-4.7
- В сегментах клиентов — `CONSUMER_CREDIT × ставка` показывает кратное изменение PD при разных режимах КС

## 3. Инфляция — «скорость инфляции стоимости денег»

**Что:** среднее повышение цен в экономике за период (год). РФ март 2026 = 5.9% годовая.

**Аналогия из IT:** инфляция — это **memory leak** в реальной стоимости денег. Каждый месяц `value -= 0.5%`.

**Связь с КС:**
- ЦБ повышает КС, чтобы охладить инфляцию (если она растёт)
- ЦБ снижает КС, чтобы стимулировать рост (если инфляция стабильна)
- Цель ЦБ РФ — 4% инфляции

**Как считается:** Росстат измеряет цены ~700 товаров и услуг в 280 городах. Считает корзину — средневзвешенный индекс.

**Где видно в Радаре:**
- L1 макро — direction (↑/↓) для шоков
- Macro snapshot в Хабе — текущее значение

## 4. ВВП — «общий объём произведённого»

**Что:** сумма всей произведённой стоимости в экономике за период. РФ 2025 = 214.3 трлн ₽.

**Аналогия:** ВВП — это **throughput системы** в byte/sec. Сколько в год экономика «выдаёт».

**Что важно:**
- Δ ВВП в % = темп роста (РФ 2025 = +1.0%)
- Падение ВВП на 2 квартала подряд = **рецессия** (формальное определение)
- Темпы роста ВВП > КС → экономика «обгоняет» стоимость денег → здорово

## 5. Курс рубля (USD/RUB) — «обменный коэффициент»

**Что:** сколько рублей за 1 доллар. На 25.04.2026 — около 75 ₽.

**Аналогия:** USD/RUB — это **exchange rate API endpoint** между рублёвой и долларовой системами.

**Драйверы курса:**
- **Цена нефти** — главный драйвер для РФ (≈60% корреляции). Brent ↑ → больше USD-выручки экспортёров → больше USD на рынке → рубль ↑
- **Ключевая ставка** — ↑ КС → выгоднее держать рубли → рубль ↑
- **Геополитика** — санкции / война → отток капитала → рубль ↓
- **Минфин-операции** — покупка/продажа валюты по бюджетному правилу

**Влияние курса на сегменты:**
- Экспортёры (нефть, металлургия): рубль ↓ → выручка в ₽ ↑
- Импортёры (розница, фарма): рубль ↓ → закупки в ₽ ↑ → инфляция
- ФЛ.Премиум с валютой: рубль ↑ → потеря в ₽-эквиваленте

## 6. Бюджет РФ — «балансовый отчёт государства»

**Структура:** доходы (НДФЛ, НДС, налог на прибыль, НДПИ, акцизы) − расходы (соц., ВПК, инфраструктура, господдержка).

**Дефицит = доходы < расходы.** В 2025 РФ имеет хронический дефицит из-за санкций и СВО.

**Нефтегазовые доходы (НГ-доходы):**
- НДПИ на нефть и газ + экспортная пошлина (отменена 2025) + дивиденды госкомпаний
- ~25-30% всех доходов федерального бюджета
- Сильно зависят от Brent и USD/RUB

**Бюджетное правило:**
- Если Urals > $59 (отсечная цена 2026) → излишки в ФНБ (Фонд национального благосостояния)
- Если Urals < $59 → дотации из ФНБ для покрытия дефицита
- Минфин **продаёт/покупает валюту** для сглаживания курса (механизм)

**Где видно в Радаре:**
- Категория **2.5 Бюджетный кризис РФ** в таксономии
- ОИВ (региональные бюджеты) зависят от федерального бюджета через МБТ (межбюджетные трансферты)

## 7. Brent / Urals — «топ commodity цен»

**Brent** — мировой бенчмарк цены нефти (Великобритания, North Sea). 25.04.2026 ≈ $95/барр после пика $119 в марте.

**Urals** — российская смесь нефти. Дисконт к Brent (cреднегодовой 2025 = $20-25 ниже Brent) из-за санкций на российский экспорт.

**Аналогия:** Brent — это **mainstream API**, Urals — **forked версия с throttling** (санкции = скорость передачи). Дисконт = «штраф за форк».

**Почему важно для риск-аналитика:**
- Brent ↑ → НГ-доходы ↑ → бюджет здоровее → ОИВ-регионы получают больше МБТ
- Urals ↓ → нефтяники теряют выручку, регионы-доноры (ХМАО, Тюмень) теряют налог
- 1 барр = 7.33 баррелей в тонне (это для перевода)

## 8. НДПИ — налог на добычу полезных ископаемых

**Что:** государство забирает % от добычи нефти/газа. Это **главный источник** НГ-доходов.

**Формула 2025 (упрощённая для нефти):**
```
НДПИ = (Urals_цена − $15) × 7.33 × курс × 0.42
       └─ за тонну ─┘   └─ FX ─┘  └ ставка ┘
```

**Топливный демпфер 2025:** в текущих условиях нефтяники **платят** в бюджет (отрицательная компенсация около −300 ₽/т) вместо получения помощи. Это редкость — обычно демпфер сглаживает разрыв между внутренней ценой топлива и мировой.

**Где видно в Радаре:**
- В коде `osl_oilgas.py` — функция `calc_ndpi_per_ton_rub()`
- Параметры `NDPI_BASE_RATE`, `DEMPFER_PAID_RUB_PER_TON`

## 9. Капитализация и долговая нагрузка

**Чистый долг (Net Debt):** общий долг − cash. Если отрицательный — компания имеет «нетто-кэш позицию» (что хорошо).

**EBITDA:** Earnings Before Interest, Taxes, Depreciation, Amortization. По сути — **операционная прибыль** до неоперационных расходов. Главный показатель «здоровья» бизнеса.

**ЧД/EBITDA (Net Debt / EBITDA):** сколько лет компании понадобится, чтобы выплатить долг из текущей EBITDA. Главная метрика **долговой нагрузки**.

**Пороги:**
- ЧД/EBITDA ≤ 2.5× — комфортная зона
- 2.5–3.5× — повышенное внимание (мониторинг)
- > 3.5× — стоп-сигнал для нового кредитования

**Примеры:**
- Норникель ЧД/EBITDA = 1.6× → ✅ комфорт
- Евраз = 5.5× → 🚫 за пределами
- УГМК = 4.92× (по дочкам, оценочно) → 🔴 высокая

**ICR (Interest Coverage Ratio):** EBITDA / процентные расходы. Сколько раз EBITDA «покрывает» проценты по долгу.
- ICR > 4× — здоровая позиция
- ICR < 1.5× — компания не зарабатывает на проценты

## 10. Кредитный рейтинг — «трастовый score»

**Аналогия:** как credit score у физлица, только для компаний/стран. Выпускают рейтинговые агентства: АКРА, Эксперт РА (РФ), S&P, Moody's, Fitch (международные — для РФ исторически).

**Шкала АКРА (от лучшего к худшему):**
- AAA (RU) — высочайшая надёжность
- AA / A — высокая
- BBB / BB — умеренная
- B / CCC — низкая
- D — дефолт

**Что меняет рейтинг:**
- Снижение → выше стоимость нового долга
- Снижение ниже B+ → закрывается доступ к капитальным рынкам

**Где видно в Радаре:**
- Сегменты M+/L клиентов — рейтинг включён в Profile

## 11. Сегментация банковских клиентов

**ФЛ (физические лица):**
- **Массовый** — 70-80% клиентов, 25-30% AUM
- **Средний** — 15-20%, 35-40% AUM
- **Премиум** — 3-5%, 20-25% AUM (премиум-карты, валюта)
- **Private Banking** — 0.5-1%, 15-20% AUM (wealth mgmt)

**SME (малый/средний бизнес):**
- **Микро** (≤ 100 млн ₽ выручки)
- **Малый** (100–500 млн)
- **Средний** (500 млн – 2 млрд)

**M+/L (крупный):**
- Mid (2–50 млрд выручки)
- Large (>50 млрд)
- Public (рейтинговые эмитенты)

В Радаре используется **многомерная сегментация**: базовый сегмент × регион × возраст × профессия × тип продукта банка × валютная экспозиция.

## 12. Supply chain контагия — «каскадный fault propagation»

**Идея:** компании связаны между собой как граф (поставщики, покупатели). Шок одной → распространяется по графу.

**Аналогия из IT:** это **microservices** + **circuit breaker**. Если один сервис упал — другие через него теряют функциональность. В худшем случае — каскадный отказ.

**Базовая модель в Радаре** — **Fialkowski et al. 2025** (arXiv:2502.17044):
- 3 слоя контагии: фирма-фирма (supply chain) + фирма-банк (займы) + банк-банк (interbank)
- Игнорирование одного слоя → недосчёт системного риска на **+28-70%**
- В Радаре — упрощённая 7×7 матрица отраслей с маркерами H/M/L

**Пример:**
- ↑ цена газа (Газпром) → ↑ себестоимость аммиака (ФосАгро/Акрон) → ↑ цена удобрений на экспорте → потенциально ↓ конкурентоспособность

## 13. Conformal Prediction — «честный confidence interval»

**Что:** статистический метод построения **prediction intervals** с заданным уровнем доверия (90%).

**Аналогия из IT:** как unit-test, который не просто говорит «test passed/failed», а возвращает **confidence score** + interval вокруг ожидаемого значения.

**Зачем:** точечный прогноз («ЛУКОЙЛ заработает 4 трлн ₽») обманчив. Нужен интервал «[3.7; 4.4] с 90% вероятностью» — тогда видно **stress-сценарий** (lower bound) и **upside** (upper bound).

**Как работает в Радаре:**
1. Прогон OSL с базовыми параметрами → predicted_base
2. 200 симуляций с шумом (±5% цены, ±3% объёмы, ±2% FX)
3. Распределение из 200 значений → quantile 5% и 95% → 90% interval

## 14. Operational Signal Layer (OSL) — «physical to financial mapping»

**Идея, разработанная в Радаре:** прогнозировать выручку компании по **физическим объёмам и ценам**, а не ждать финансового отчёта.

**Формулы по отраслям:**
- Нефтегаз: `Q_oil × Urals × курс × (1 - НДПИ_share) + downstream_revenue`
- Металлургия: `Q_steel × FOB_цена × курс` (для чёрной)
- Розница (маркетплейсы): `GMV × take_rate`
- Энергетика: `Q_kWh × тариф`

**Преимущество:** **2-3 месяца форы** перед публикацией МСФО — за это время операционные данные (квартальные production reports) уже опубликованы.
