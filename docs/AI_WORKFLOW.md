# AI-воркфлоу — как ИИ применялся в проекте

Радар использует ИИ на **двух уровнях**:

1. **Внутри продукта** — слой L0 это multi-agent LLM-пайплайн (см. ниже).
2. **В самой разработке** — сбор данных, верификация, ревью, тесты и рефакторинг шли через
   AI-агентов, но каждый шаг закрыт **человеческим гейтом**.

Ключевой принцип — **ИИ под контролем**: агент предлагает, человек решает, а каждый вывод
привязан либо к публичному источнику, либо к падающему тесту. Это отличает применение ИИ от
«спросил чат и вставил».

---

## 1. ИИ в продукте — слой L0

- **5 агентов:** `Classifier → Context-RAG → Backtest-Analog → Impact → Summary`. Промпты
  живут как markdown и парсятся в рантайме (`extract_prompt`); state течёт как растущий JSON.
- **RAG:** SQLite + `sqlite-vec` по корпусу `_Анализы/`; эмбеддер переключается флагом
  (`RADAR_RAG_USE_ST`: TF-IDF ↔ e5).
- **Измерен, а не задекларирован.** Gold-set N=15, живой прогон через API:
  Haiku 4.5 → main 93% / подкатегория **100%**; Sonnet 4.6 → **100% / 100%**. Метод и оговорки —
  [`L0_EVAL.md`](L0_EVAL.md).
- **ИИ проверяет ИИ.** Живой eval вскрыл прод-баг (плейсхолдеры промпта не совпадали с ключами
  подстановки → новость не инжектилась; dry-run это скрывал) → фикс + офлайн regression-тест-страж.

---

## 2. ИИ в разработке — паттерны (с реальными примерами)

| Паттерн | Что делал ИИ | Человеческий гейт | Что реально поймал |
|---|---|---|---|
| **Агентный сбор данных** | извлекал выручку / цены / объёмы из публичных IR- и МСФО-релизов | сверка с первоисточником | — |
| **`/doublecheck` + `/fact-check`** | верифицировал каждую ячейку панели | ручное подтверждение спорных | **6 битых ячеек** агрегаторов (сдвиги рядов); деконсолидация ЛУКОЙЛ-2025 (разрыв ≠ падение) |
| **Ревью-субагент** (гейт ≥ 9/10) | ревьюил каждую DS-фазу на переобучение и overclaim | фаза не принимается при < 9/10 | раздутые интерсепты энергетики (8% → **честные 12%**); overclaim про плотность панели |
| **AI-assisted инженерия** | тесты, CI, рефакторинг, форматирование | 269 тестов + жёсткие CI-гейты | latent-баг инъекции state между агентами L0 |
| **ИИ оценивает ИИ** | eval L0-классификатора на gold-set | regression-тест на контракт промптов | плейсхолдер-десинк промпта L0 |

Каждая строка — это конкретный пойманный дефект, а не «ИИ помог». Именно ошибки, вскрытые
агентным ревью и верификацией, — главный результат этого воркфлоу.

## 3. Guardrails — что ИИ **не** делал

- **Не выдумывал числа.** Каждая цифра — публичный источник; пробелы **документированы, не
  занулены** (палладий ≈ 30% выручки одного эмитента — нет в открытых годовых; HRC spot —
  платный, заменён прокси iron-ore).
- **Не подгонял под красивый результат.** Директива «удалять плохих клиентов, на которых модель
  не ложится» (НКНХ, Уралхим — нет отчётности) вместо подгонки; тесты-стражи против «тихой
  подгонки» интерсептов/метрик.
- **Не принимал продуктовых решений вслепую.** Гипотезы проверялись на данных: девелопмент как
  5-я валидированная отрасль — прогнан walk-forward’ом и **отклонён** (IFRS-15 over-time
  размазывает выручку по backlog → structural 26.8% проиграла persistence 19.1%), а не добавлен
  ради числа. Разбор — [`DEVELOPERS_EVALUATION.md`](DEVELOPERS_EVALUATION.md).
- **Не оставался непроверяемым.** Любой вывод — либо публичный источник, либо падающий тест,
  либо запись в DS-отчёте.

---

## 4. Прослеживаемость

Итог AI-воркфлоу — не «чёрный ящик, который написал ИИ», а проверяемый артефакт:
**269 тестов** (в т.ч. страж-тесты контрактов и анти-leakage), **CI из 4 джобов**
(matrix + e2e smoke + Docker clean-clone + security), DS-отчёты по отраслям и манифест покрытия.
Любой шаг открывается и проверяется руками.
