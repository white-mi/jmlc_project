# Макро-радар

### Новость → макро РФ → отрасль → **кредитный риск сегмента банка**
### …на **2–3 месяца раньше**, чем эффект дойдёт до отчётности

## Проблема

<div class="cols">
<div>

Классический стресс-тест скорит каждого заёмщика **в изоляции**.

Он пропускает доминирующий механизм кризиса: один макро-драйвер бьёт по
**коррелированной группе** заёмщиков сразу — через общие цепочки поставок.

<span class="sub">Fialkowski et al., 2025 · arXiv:2502.17044</span>

</div>
<div style="text-align:center">

<span class="big">+28…70%</span>
<br><span class="mid">системного риска</span>
<br><span class="sub">в слепой зоне «по одному»</span>

</div>
</div>

<!--
ЗАМЕТКА (40 сек): Банки считают риск по каждому заёмщику отдельно. Но в кризис

## Идея: один прослеживаемый путь

<div class="chain">

новость → макро-состояние РФ → межотраслевой каскад → сегмент клиентов банка

</div>

<br>

Каждое число привязано к **каналу передачи** и **публичному источнику** —
аналитик может *защитить* вывод, а не доверять чёрному ящику.

<div class="card">

**Выход:** Δ вероятности дефолта / Δ спроса / Δ оттока по 18 сегментам клиентов.

</div>

<!--

## Архитектура — 5 слоёв

| Слой | Что делает |
|---|---|
| **L0** Фильтр новостей | 5 LLM-агентов + RAG → 27 подкатегорий шоков |
| **L1** Макро-состояние | РФ-CAI · EPU · режим ключевой ставки |
| **▶ L1.5** Операционный сигнал | **выручка = объёмы × цены × FX, раньше МСФО** + conformal |
| **L2** Spillover | матрица отраслей 7×7 (Fialkowski) · кредитный канал |
| **L3** Сегменты | ΔPD / Δспрос / Δотток · 18 сегментов · 5 каналов |

<span class="sub">▶ L1.5 — ядро DS-валидации этого доклада.</span>

## Что отличает: Data Science уровня, а не демо

**4 отрасли** проработаны на **реальных панелях FY2021–2025**
<span class="sub">(только публичные МСФО + биржевые цены — никаких данных банка)</span>:

- честная **out-of-sample** walk-forward — expanding-window, сплит по времени
- **split-conformal** интервалы (finite-sample)
- наивные бейзлайны (persistence / issuer-mean) + тест **Diebold–Mariano**
- **анти-leakage аудит** на каждом шаге

<div class="card">

Это отделяет «обучил модель на одной точке» от **проверяемой валидации**.

</div>

## Интеллектуальная честность = зрелость

Не «красивые цифры», а **воспроизводимая правда**:

- `/doublecheck` + `/fact-check` поймали **6 битых ячеек** агрегаторов
  (сдвиги рядов) и структурный разрыв — деконсолидацию ЛУКОЙЛа
- независимое ревью убрало **раздутые интерсепты** в энергетике:
  «красивые» 8% → **честные 12%**
- каждое ограничение **задокументировано**; тесты-стражи против «тихой подгонки»

<div class="card">

Готовность сказать «модель не лучше наивной» — это и есть DS-зрелость.

## Честные ограничения (фичи, не баги)

- DS глубоко на **4 из 7** отраслей; малый N → выводы с DM / skill, не «доказанная точность»
- **L3 не калиброван** на данных банка (`confidence=low`, экспертные приоры)
- **×1.30** spillover-усилитель — эвристика Fialkowski, не калибрована на РФ-шоках
- теплосегмент энергетики — вне структурной модели

<span class="sub">Всё это записано в DS-отчётах. Знаем границы — значит контролируем их.</span>
