# L0-eval — точность классификатора шоков (Agent 1)

> **Что это.** Живой прогон L0-классификатора (Agent 1) на gold-set из 15
> синтетических **публичных** новостей. Проверяем, правильно ли агент раскладывает
> новость по таксономии шоков (5 категорий / 27 подкатегорий) — прежде чем
> результат уйдёт в L1–L3.

> **Что это НЕ.** Не статистически значимый бенчмарк. N=15 — это **sanity-check**,
> а не leaderboard. Метки проставлены вручную по таксономии; новости синтетические
> (клиентских/портфельных данных банка нет — [принцип изоляции](../CLAUDE.md)).

Источник чисел — [`_tools/data/l0_eval_results.json`](../_tools/data/l0_eval_results.json)
(собран из сырых прогонов `output/l0_eval/`, которые в `.gitignore`).
Gold-set — [`_tools/data/l0_gold_set.json`](../_tools/data/l0_gold_set.json).

---

## Метод

- **Промпт — продакшн, не отдельный.** Harness (`_tools/eval_l0_classifier.py`) берёт
  ровно тот промпт, что уходит в прод: `orchestrator.extract_prompt(agent_1_classifier.md)`
  + `orchestrator.fill_prompt(...)`. Никакого «облегчённого» промпта под тест.
- **Таксономия инлайнится.** В проде режим `--llm-mode cli` читает справочник таксономии
  как файл (инструментом Claude Code). Голый Anthropic API так не умеет, поэтому harness
  инжектит компактную 27-подкатегорийную таксономию в промпт — это воспроизводит то,
  что cli-агент получает из файла.
- **Метрики:** accuracy по `main_category` (5 классов) и `subcategory` (27 классов).
  Подкатегория объективно труднее. Два намеренно **пограничных** кейса (`boundary=true`:
  g14 = ожидаемое/уже-в-ценах снижение КС → 5.2 vs 4.2; g15 = календарный артефакт →
  5.4) считаются отдельной колонкой.
- **Стоимость** берётся из `response.usage` (реальные токены) × опубликованные тарифы.

## Результаты

| Модель | main-cat (5-класс.) | subcategory (27-класс.) | subcat без boundary | стоимость |
|---|:-:|:-:|:-:|--:|
| **Haiku 4.5** | 93.3 % (14/15) | **100 % (15/15)** | 100 % (13/13) | $0.055 |
| **Sonnet 4.6** | **100 % (15/15)** | **100 % (15/15)** | 100 % (13/13) | $0.193 |

Оба прогона: **$0.25** суммарно (25 471 входных токенов на модель).

**Интерпретация единственного промаха.** Haiku ошибся ровно на одном пункте — g14,
намеренно пограничном: «ЦБ восьмой раз подряд снизил ставку ровно как заложил
консенсус; облигации не отреагировали — решение полностью в ценах». Haiku **верно**
поставил подкатегорию **5.2** (ожидаемое/в-ценах), но `main_category` отнёс к **4**
(ставка), а не к **5** (информационный шум). Это тонкий, защитимый near-miss на
границе «формально ставочное событие vs по сути шум», а не ошибка сортировки. Sonnet
на этом же кейсе дал и 5, и 5.2. Дешёвая модель уже насыщает 27-классовую разметку —
поэтому Opus на этом gold-set не гоняли (потолок достигнут, экономия бюджета).

## Баг, который нашёл этот eval (и почему это ценно)

Первый прогон дал **0 %** — `pred=None` на всех 15 пунктах; модель отвечала
«готов, присылайте данные». Причина оказалась реальным латентным дефектом прода:

- В `agent_1_classifier.md` плейсхолдеры были `<текст>` / `<URL/издание>` /
  `<YYYY-MM-DD>`, а `orchestrator.fill_prompt()` подставляет по ключам
  `НОВОСТЬ` / `ИСТОЧНИК` / `ДАТА`. Токены не совпадали → **текст новости
  никогда не инжектился** в реальном LLM-режиме.
- Почему тесты это не ловили: `--llm-mode dry-run` возвращает захардкоженную
  заглушку и **не вызывает** `fill_prompt`. Баг был невидим офлайн.

**Что сделано:**
1. Синхронизированы плейсхолдеры Agent 1 (+ таксономия в промпте приведена к
   актуальным 27 подкатегориям вместо 18).
2. Тем же проходом закрыты аналогичные пропуски инъекции состояния в Agent 2–5
   (`<STATE_JSON>`, `<RAG_ANALOGS>`, `<BIFURCATION>`) и обрыв промпта Agent 5
   вложенным ```-fence (`extract_prompt` возвращал 377 символов из 3277 —
   вложенные блоки переведены на `~~~`).
3. Добавлен офлайн regression-тест [`tests/test_l0_prompt_contract.py`](../_tools/tests/test_l0_prompt_contract.py):
   каждый ключ, который orchestrator подставляет, обязан присутствовать как
   `<KEY>` в промпте агента, а блок `## Промпт` — извлекаться целиком. Этот класс
   рассинхрона больше не пройдёт в прод молча.

Мораль: **живой eval окупился не цифрой accuracy, а найденным багом.** Без него
multi-agent L0 «работал» в dry-run и был сломан в реальном LLM-режиме.

## Честные оговорки

- **N=15, sanity-check.** Не заявляем статистической значимости и не экстраполируем
  на генеральную совокупность новостей.
- **In-sample по духу.** Gold-метки и таксономия писались одной командой; это проверка
  «промпт делает то, что задумано», а не независимый hold-out.
- **Синтетические новости.** Переформулированы из примеров таксономии; реальных
  клиентских данных нет.
- **Boundary-кейсы вынесены отдельно** — именно там живёт содержательная
  неоднозначность, и мы её не прячем в общий процент.

## Воспроизведение

```bash
cd _tools
export ANTHROPIC_API_KEY=...            # без ключа — graceful exit, CI-safe
python eval_l0_classifier.py --model haiku      # ~$0.05
python eval_l0_classifier.py --model sonnet     # ~$0.19
# сырые результаты → output/l0_eval/<model>_results.json (в .gitignore)

# офлайн (без API) — regression на контракт промптов:
python -m pytest tests/test_l0_prompt_contract.py -v
```
