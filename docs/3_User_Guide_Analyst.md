# 📕 User Guide — для аналитика

> [[../Макро-радар — Хаб|← Хаб]] · [[1_Macro_Knowledge_for_IT|← Макро-знания]] · [[2_Tech_Knowledge_for_IT|← Тех-знания]] · [[4_Tech_Architecture|→ Тех. документация]]

Этот документ — для риск-аналитика, который **использует** «Макро-радар». Здесь — что Радар умеет, как формулировать запросы и какие сценарии покрыты.

---

## 1. Что делает Радар (за 30 секунд)

Радар берёт **новость** (из СМИ, Reuters, ТАСС, ЦБ-релиз и т.д.) и за 2-5 минут даёт:

1. **Классификацию** шока (война / кризис / климат / ставка ЦБ / шум) с силой 0-100
2. **Макро-эффект** на ключевые индикаторы (КС, Brent, USD/RUB, инфляция, дефицит бюджета)
3. **Отраслевой эффект** на 7 отраслей (нефтегаз, металлургия, химия, фарма, энергетика, розница, ОИВ)
4. **Клиентский эффект** на сегменты ФЛ / SME / M+/L
5. **Прогноз выручки** ключевых эмитентов с **90% prediction interval** (через OSL + Conformal)
6. **Триггеры мониторинга** на 1-3 месяца
7. **Вердикт «громкий шум vs тихий-критичный»** — главная ценность Радара

## 2. Как устроена модель экономики (4-слойная архитектура)

```
┌─────────────────────────────────────────────────────┐
│ L0 — News Filter                                     │
│   Новость → классификация по таксономии (5 кат.)    │
└────────────────────┬────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────┐
│ L1 — Macro State                                     │
│   Какие макро-индикаторы изменятся?                  │
│   КС / Brent / USD/RUB / инфляция / дефицит         │
└────────────────────┬────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────┐
│ L1.5 — Operational Signal Layer (★)                 │
│   Прогноз выручки эмитентов по фактам производства  │
│   через OSL-формулы                                  │
└────────────────────┬────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────┐
│ L2 — Industry Spillover                              │
│   Карта 7×7 отраслей: как шок передаётся между     │
│   ними через supply chains                           │
└────────────────────┬────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────┐
│ L3 — Client Behavior                                 │
│   ΔPD / Δdemand / Δchurn для микросегментов:        │
│   ФЛ × регион × возраст × профессия × продукт       │
└─────────────────────────────────────────────────────┘
```

**Важно:** L3 — channel-decomposition **реализован** (5 каналов передачи × 18 сегментов, профили регионов), но **НЕ калиброван на данных банка**. Поле `confidence` — честное поле данных со значением `'low'` (экспертные оценки, не валидация на реальных PD/портфеле).

## 3. Как формулировать запрос к Claude

### 3.1. Базовый шаблон

Открой `Макро-радар/Промпт — Анализ новости.md`, скопируй блок `ВХОД:` и заполни три поля:

```
НОВОСТЬ: <вставь полный текст новости>
ИСТОЧНИК: <URL или название издания>
ДАТА: <YYYY-MM-DD>
```

Пример:
```
НОВОСТЬ: ЦБ РФ снизил ключевую ставку с 15% до 14.5% годовых на заседании 24 апреля 2026...
ИСТОЧНИК: https://cbr.ru/press/event/?id=12345
ДАТА: 2026-04-24
```

### 3.2. Опциональные поля

```
КОНТЕКСТ: <если есть фон, которого нет в самой новости>
ФОКУС: <если интересует только один сегмент, например "ФЛ.Массовый"
        или одна отрасль, например "только нефтегаз">
```

### 3.3. Когда использовать Multi-Agent vs одиночный промпт

| Тип шока | Что использовать |
|---|---|
| Сила ≤ M (≤30/100), категория простая | **Одиночный промпт** (быстрее, ~1 мин) |
| Сила ≥ M, композитный шок (>1 категория) | **Multi-Agent** (5 агентов, ~3-5 мин, точнее) |
| Касается M+/L клиентов или системные эффекты | **Multi-Agent** |
| Информационный шум (риторика без действия) | **Одиночный промпт** + сразу пометить категорию 5 |

## 4. Какие сценарии покрыты

<details>
<summary><b>📂 Категория 1 — Война / Геополитика (раскрой)</b></summary>

#### 1.1 Военный конфликт — старт/эскалация
- **Что покрывается:** удары/война/мобилизация — глобальные триггеры роста Brent и оттока капитала
- **Эталон:** Иран-Израиль 02.2026, Hormuz crisis
- **Сила типичная:** H (70-100)
- **Как влияет:** Brent ↑↑, USD/RUB → волатильность, ФЛ.Премиум × HARD_FX → защитные активы
- **Триггеры мониторинга:** транзит танкеров, страховые премии, переговоры

#### 1.2 Военный конфликт — деэскалация/завершение
- **Что покрывается:** прекращение огня, заморозка конфликта
- **Сила типичная:** M (40-60)
- **Как влияет:** Brent ↓, нормализация логистики, отток в спекулятивные активы

#### 1.3 Точечные удары / диверсии
- **Что покрывается:** дроны на НПЗ, диверсии на инфраструктуре
- **Эталон:** Туапсе 16-20.04.2026, Татарстан 18-25.04.2026
- **Сила типичная:** L–M (20-60), накопительный эффект → M
- **Как влияет:** Прямой удар по отрасли, локальный шок

#### 1.4 Новый санкционный пакет
- **Что покрывается:** ЕС / США / Великобритания пакеты против РФ
- **Эталон:** 20-й пакет ЕС 23.04.2026 (урезанный)
- **Сила типичная:** M-H (50-90)
- **Как влияет:** Закрытие финрынков, дисконт Urals, юанизация

#### 1.5 Прокси-конфликт / расширение фронта
- **Что покрывается:** конфликты в Африке, Йемене, Тайване
- **Сила типичная:** L-M (20-60)

</details>

- **Эталон:** Colonial Pipeline 2021, Туапсе 04.2026 (комбо)

- **Эталон:** Ущерб портам (Новороссийск, Сахалин)
- **Сила:** L-M (20-60)

#### 3.3 Землетрясение / геофизика
- **Эталон:** Турция 2023, цунами
- **Сила:** M-H (50-80)

#### 3.4 Экстремальная жара / засуха
- **Эталон:** Лето 2010 РФ, засуха 2022 Поволжье

- **Эталон:** Сибирские пожары 2019, Туапсе 20-24.04.2026

<details>
<summary><b>📂 Категория 4 — Ставка ЦБ / Монетарная политика</b></summary>

#### 4.1 Резкое повышение КС (≥+200 бп)
- **Эталон:** КС 9.5% → 20% в 02.2022, 16% → 21% в 12.2024
- **Сила:** H (70-100)
- **Канал:** Кредитный канал, инвестиции, потребление, депозиты

#### 4.2 Резкое снижение КС (≥−200 бп)
- **Эталон:** КС 21% → 15% в 03.2026, 20% → 8% в 2022
- **Сила:** M-H (60-90)
- **Канал:** Восстановление спроса на кредит, ослабление рубля

#### 4.3 Ястребиная риторика без изменения
- **Эталон:** «Готовы повысить, если инфляция не замедлится»
- **Сила:** L-M (20-50)

#### 4.4 Голубиная риторика без изменения
- **Сила:** L-M (20-50)

#### 4.5 Изменение нормативов / макропруденциальные меры
- **Эталон:** Надбавки на крупные корп. кредиты с высокой долговой нагрузкой
- **Сила:** M (40-70)

#### 4.6 Инфляционный сюрприз (выше/ниже ожиданий)
- **Эталон:** Инфляция 9% при ожиданиях 7%
- **Сила:** M (40-70)

#### 4.7 Валютная интервенция
- **Эталон:** Минфин-операции по бюджетному правилу, обязательная конвертация
- **Сила:** M (40-70)

</details>

<details>
<summary><b>📂 Категория 5 — Информационный шум (новое v1.1)</b></summary>

> ⚠️ Категория для отсева, чтобы не тратить ресурсы Радара на шум.

#### 5.1 Риторика официальных лиц без действия
- **Эталон:** Решетников «ресурсы экономики кончились» 23.04.2026
- **Канал:** Психологический эффект на дни; угасает быстро
- **Action:** Радар выдаст краткую пометку, не полный анализ

#### 5.2 Ожидаемое решение / consensus в СМИ
- **Эталон:** Восьмое подряд снижение КС
- **Канал:** Всё уже в ценах
- **Action:** Краткая пометка

#### 5.3 Громкая угроза / ультиматум без эскалации
- **Эталон:** Угрозы санкций без конкретики
- **Action:** Мониторинг — если будет действие, перейти в категорию 1.4

#### 5.4 Календарный артефакт
- **Эталон:** ВВП Q1 −1.5% (фактически меньше рабочих дней)
- **Канал:** Ложный сигнал

</details>

## 5. Какие сегменты клиентов покрыты

<details>
<summary><b>📂 ФЛ — Физические лица (4 базовых × 5 размерностей)</b></summary>

### Базовые сегменты
1. **ФЛ.Массовый** — остаток < 100к ₽, доход < 80к ₽/мес
2. **ФЛ.Средний** — 100к-1М ₽, доход 80-250к ₽/мес

**Регион (двухуровневый, новое v1.2):**
- Макро: MSK_SPB / DONOR / INDUSTRIAL / SOUTH_CAUCASUS / FAR_EAST / SIBERIA / CENTRAL / RURAL
- Микро: TOURIST_RESORT / URBAN_INDUSTRIAL / MONOTOWN / AGRICULTURAL_RURAL / CAPITAL_DIVERSIFIED

**Возраст:**
- YOUNG (18-25), MILLENNIAL (26-35), MID_CAREER (36-50), PRE_RETIRE (51-65), RETIREE (65+)

**Профессия:**
- GOV_BUDGET, INDUSTRY_WORKER, SERVICES, IT_FREELANCE, SELF_EMPLOYED, MILITARY_CONTRACT, CORP_EXEC, PENSIONER

**Тип продукта банка:**
- MORTGAGE_FIXED / MORTGAGE_FLOAT / CONSUMER_CREDIT / DEPOSITOR / INVESTOR / SALARY_ONLY / BNPL_USER / PREMIUM_VALUTA

**Валютная экспозиция:**
- RUB_ONLY / MIXED / HARD_FX

## 7. Что радар НЕ покрывает (явно)

- ❌ **Real-time мониторинг 24/7** — пока ручной запуск
- ❌ **Внутренние данные банка** — Слой 3 (channel-decomposition) реализован, но не калиброван на реальных PD/портфеле; confidence = 'low'
- ❌ **Точные числовые прогнозы клиентских метрик** — только направление и сила
- ❌ **Прогнозы валютных курсов / commodity prices** — Радар реагирует на изменения, а не предсказывает их
- ❌ **Гипотетические события «что если»** — нужны реальные новости
- ❌ **Микро-уровень** (отдельная компания SME) — только агрегаты

## 8. Регулярные сценарии использования

### 8.1. Еженедельный дайджест
Раз в неделю — собирай 7-10 ключевых новостей за период и прогоняй через **Компактный multi-agent** (карточки + сводная таблица). Эталон: `_Анализы/2026-04-25 — Калибровочный дайджест Multi-Agent (18-25 апреля).md`

### 8.2. Анализ конкретной новости
Когда выходит важное событие (ЦБ, санкции, ЧС) — **полный multi-agent** (5 агентов). Эталон: `_Анализы/2026-04-20 — Туапсе экологическая катастрофа после атак.md`

### 8.3. Прогноз ΔRevenue эмитента
Если изменился макро-параметр (Brent, USD/RUB) и нужно срочно понять что с эмитентами — `python conformal_prediction.py --industry <отрасль>` за 5 секунд даст 90% interval по всем 28 эмитентам.

### 8.4. Регулярная калибровка
Раз в квартал после публикации МСФО — обновить ACTUAL_REVENUE и запустить:
```bash
python osl_calibrator.py --module all
```
Все модели обновлены.

❌ «Что будет с экономикой?» (слишком общее, нет конкретного шока)

❌ «Какие клиенты пострадают от инфляции?» (нужен конкретный сценарий: что вызвало инфляцию?)

## 10. Часто задаваемые вопросы

<details>
<summary><b>Что делать если новость попадает в несколько категорий?</b></summary>

Указать главную + composite. Радар делает анализ по главной, но учитывает дополнительные через каналы L1/L2.
</details>

<details>
<summary><b>Что делать если Conformal интервал не покрывает factual?</b></summary>

Это сигнал что **base модель некалибрована**. Запустить `python osl_calibrator.py --module <отрасль>` и пересчитать.
</details>

<details>
<summary><b>Как добавить нового эмитента в OSL?</b></summary>

См. `_tools/CALIBRATION_GUIDE.md` секция «Сценарий 3 — Добавлен новый эмитент».
</details>

<details>
<summary><b>Что делать если Радар выдаёт невнятный анализ?</b></summary>

Возможные причины:
1. Текст новости короткий или неоднозначный → дай больше контекста через `КОНТЕКСТ:`
2. Категория шока не покрыта в таксономии → переформулируй или укажи custom-категорию
3. Эмитент не в OSL-базе → добавь в соответствующий osl_*.py
</details>

<details>
<summary><b>Какова частота калибровки?</b></summary>

- Базовая: раз в квартал (после публикаций МСФО)
- Срочная: при выходе новости с шоком ≥ M по 1.4 (санкции) или 4.1/4.2 (резкое движение КС)
- Drift check: еженедельно через `python osl_calibrator.py --module drift`
</details>
