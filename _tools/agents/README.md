# Multi-Agent LLM Pipeline для L0

> [[../../Макро-радар — Хаб|← Хаб]] · [[../../Свежие модели — 12 кандидатов v1|← Идея 3 + 11]]

Декомпозиция текущего одиночного промпта L0 на **5 специализированных агентов**, работающих в pipeline. Архитектура основана на **P1GPT** (arXiv:2510.23032) и **MarketSenseAI 2.0** (arXiv:2502.00415).

---

## Зачем разбивать на агентов

Текущий одиночный промпт L0 (`Промпт — Анализ новости.md` v1.1) — это монолит, который Claude выполняет за один проход. Проблемы:
- **Высокий риск галлюцинаций** на сложных композитных шоках (война × санкции × курс).
- **Нет явной точки RAG** — невозможно подмешать исторический контекст.
- **Одна оценка** для всех 5 шагов — нельзя независимо проверить каждый.
- **Нет итеративного улучшения** — если последний шаг даёт спорный ответ, нет возврата к предыдущему.

Multi-agent решает все 4 проблемы.

## Архитектура pipeline

```
                       Новость + контекст
                              │
                              ▼
          ┌─────────────────────────────────────┐
          │ Agent 1 — Классификатор (L0.1)      │
          │ Категория шока + сила + временной    │
          │ профиль + накопительный индикатор    │
          └────────────┬────────────────────────┘
                       │ classified_event JSON
                       ▼
          ┌─────────────────────────────────────┐
          │ Agent 2 — Контекст-RAG (L0.2)       │
          │ Поиск релевантного контекста        │
          │ (отраслевые отчёты, источники)      │
          └────────────┬────────────────────────┘
                       │ classified_event + context
                       ▼
          ┌─────────────────────────────────────┐
          │ Agent 3 — Бэк-тест-аналог (L0.3)    │
          │ Поиск исторических аналогов через   │
          │ news embeddings (cosine similarity) │
          └────────────┬────────────────────────┘
                       │ + historical_analogs
                       ▼
          ┌─────────────────────────────────────┐
          │ Agent 4 — Импакт (L1+L2+L3)         │
          │ Применение карты L1/L1.5/L2/L3 с    │
          │ учётом контекста и аналогов         │
          └────────────┬────────────────────────┘
                       │ + impact_analysis
                       ▼
          ┌─────────────────────────────────────┐
          │ Agent 5 — Сводка (Output)           │
          │ Громкость vs Импакт + триггеры +    │
          │ итог в формате дайджеста            │
          └────────────┬────────────────────────┘
                       │
                       ▼
              Финальный анализ-документ
```

## Преимущества vs одиночный промпт

| Аспект | Одиночный промпт v1.1 | Multi-Agent Pipeline |
|---|---|---|
| Риск галлюцинации | Средний (один проход) | **Низкий** (5 проверок) |
| RAG-контекст | Нет | **Есть** (Agent 2) |
| Бэк-тест на аналогах | Ручной поиск | **Автоматический** (Agent 3) |
| Итеративность | Нет | **Есть** (можно вернуть с правками) |
| Прослеживаемость | Один длинный ответ | **5 промежуточных артефактов** |
| Стоимость токенов | 1 промпт × длинный | 5 промптов × средние |
| Время выполнения | 30-60 сек | **2-5 мин** (последовательно) |

**Trade-off:** Multi-agent дороже по токенам в 1.5-2× и медленнее в 4-5×, но даёт **существенно более качественный** анализ для шоков сложности M+.

**Selective use:** для шоков L (информационный шум) — оставляем одиночный промпт; для шоков M+ — multi-agent.

## Как запустить

**Автоматически (реальный orchestrator, v0.9):**

```bash
python orchestrator.py --news-file news.txt --source "ТАСС" --date 2026-06-15 --llm-mode cli
# --llm-mode: cli (через `claude -p`, подписка) | sdk (Anthropic API) | dry-run
```

`orchestrator.py` прогоняет все 5 агентов на одной новости, подмешивает RAG-аналоги и собирает финальный отчёт.

**Вручную:** в Claude Code вызывать каждого агента последовательно через **Agent tool** с указанием соответствующего промпта, передавая промежуточные результаты как контекст следующему.

## Состояние

**v1.0 промпты (2026-04-25):** ✅ промпты агентов написаны как markdown-инструкции, готовы к использованию.

**RAG — ✅ реализован (v0.9, июнь 2026):** реальный код в `rag/*.py` (`init_db`, `embeddings`, `find_analogs`, `index_news`). Локальная БД эмбеддингов всех проанализированных новостей (SQLite + sqlite-vec). Эмбеддер по умолчанию — TF-IDF (всегда доступен, без тяжёлых моделей); опционально нейросетевой e5-small (`intfloat/multilingual-e5-small`, 384d) через env `RADAR_RAG_USE_ST=1` + reindex, с graceful fallback на TF-IDF (не FinBERT по умолчанию). Индексация: `index_single` (UPSERT, не стирает БД) + `index_all`.

**Orchestrator — ✅ реализован (v0.9, июнь 2026):** `orchestrator.py` — автозапуск pipeline на новости (`--llm-mode cli|sdk|dry-run`). Ранее планировался как v2.0.

**Впереди:** out-of-sample калибровка RAG, cross-encoder rerank, расширение на новости из открытых источников (не только наши анализы).
